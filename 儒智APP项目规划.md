# 儒智 APP 项目规划文档

## 1. 项目概述

### 1.1 项目背景

"儒智"是一款融合传统儒家经典与现代科技的移动应用，旨在让古代智慧在现代社会焕发新生。通过 AI、OCR、知识图谱等技术，使用户能够便捷地接触、理解和应用传统文化精髓。

### 1.2 项目目标

- 构建领先的古籍数字化与智能解析平台
- 打造沉浸式儒家经典学习体验
- 促进传统文化的现代传承与创新应用
- 建立兼具学术严谨性与普适性的文化产品

### 1.3 目标用户群

- 传统文化爱好者与研究者
- 学生与教育工作者
- 文化创意从业者
- 对中国传统哲学有兴趣的普通用户

## 2. 功能规范

### 2.1 古籍智能识别系统

#### 2.1.1 图像获取与预处理

- **拍摄与导入**：支持拍摄实体书籍、碑拓或导入已有图像
- **图像增强**：自动校正透视变形、优化光线条件、去除噪点
- **图像分割**：识别页面布局，分离正文、批注、图像

#### 2.1.2 文字识别引擎

- **多模态 OCR 系统**：
  - 繁体字识别（准确率目标 ≥92%）
  - 异体字映射（支持常见 3000 个异体字）
  - 残缺字推断（基于上下文语义）
- **生僻字处理**：
  - 建立 20 万生僻字索引库
  - 提供字形结构分析
  - 集成《汉语大字典》、《中华字海》等权威释义

#### 2.1.3 版本对照系统

- **自动匹配算法**：基于文本相似度计算匹配不同版本
- **差异标注**：采用 NLP 语义分析高亮显示关键差异
- **版本库**：初期集成 10 种主要版本的《论语》、《孟子》

### 2.2 《孟子》知识图谱系统

#### 2.2.1 多维关系网络

- **人物关系图谱**：
  - 核心人物节点：孟子及其弟子、政治人物等 50 个主要角色
  - 关系类型：师生、政治、辩论对手等
  - 交互方式：拖拽式探索、关系路径追踪
- **概念溯源网络**：
  - 核心概念节点：性善论、王道、仁政等 30 个核心概念
  - 关联资源：原文出处、历史事件、演化过程

#### 2.2.2 智能问答引擎

- **查询类型**：
  - 事实型（"孟子何时见梁惠王"）
  - 观点型（"孟子对战争的看法"）
  - 应用型（"如何将孟子思想应用于现代管理"）
- **回答结构**：
  - 直接答案 + 原文出处
  - 拓展资料推荐
  - 相关概念链接

### 2.3 《论语》AI 智能体

#### 2.3.1 虚拟导师系统

- **角色选择**：
  - 学术型：以朱熹、程颢程颐等注家风格解读
  - 通俗型：以现代语言、生活实例诠释
  - 辩论型：提供不同学派视角的解读
- **教学模式**：
  - 系统学习：章节式课程设计
  - 情境应用：基于用户问题提供场景化解读
  - 互动问答：深入探讨经文含义

#### 2.3.2 思想实验室

- **价值观测试**：基于用户回答分析其思想倾向
- **经典情境重现**：设计基于经典故事的决策场景
- **辩论训练**：AI 扮演不同学派角色与用户进行思想交锋

### 2.4 传统文化活化工具

#### 2.4.1 文言文转换引擎

- **转换级别**：
  - 直译级：保留原文结构的现代汉语
  - 意译级：重构句式的通俗表达
  - 应用级：提取核心思想并联系现代场景
- **用户群体适配**：
  - 学术研究版
  - 教育教学版
  - 青少年阅读版

#### 2.4.2 文化创作辅助

- **经典风格仿写**：
  - 提供多种文体模板：论、说、策等
  - 智能推荐典故、用词
- **现代议题儒学解析**：
  - 分析现代问题的传统思想视角
  - 生成符合儒家逻辑的分析框架

#### 2.4.3 AR 文化体验

- **文物复原**：
  - 识别文物图像并生成 3D 模型
  - 提供历史背景与文化意义解读
- **礼乐复原**：
  - 基于历史研究还原音乐演奏
  - 可视化古代礼仪场景

## 3. 技术架构

### 3.1 系统架构

```
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│  客户端层     │     │   服务层      │     │   数据层      │
├───────────────┤     ├───────────────┤     ├───────────────┤
│ - iOS App     │     │ - API网关     │     │ - 文本库      │
│ - Android App │<--->│ - 微服务集群  │<--->│ - 图谱数据库  │
│ - Web界面     │     │ - AI推理服务  │     │ - 媒体资源库  │
└───────────────┘     └───────────────┘     └───────────────┘
```

### 3.2 数据架构

#### 3.2.1 核心数据库

- **古籍文本库**：
  - 类型：ElasticSearch 全文检索 + MongoDB
  - 规模：初期收录先秦两汉经典文献约 500 万字
  - 元数据：版本、朝代、作者、注释者等
- **知识图谱库**：
  - 类型：Neo4j 图数据库
  - 节点规模：初期 10,000 个概念节点，50,000 个关系边
  - 示例结构：
    ```
    (孟子)-[提出]->(性善论)-[影响]->(宋明理学)
    (孟子)-[师从]->(子思)-[师从]->(曾子)-[师从]->(孔子)
    ```

#### 3.2.2 媒体资源库

- **图像资源**：古籍影像、文物图片、人物画像
- **3D 资源**：文物模型、礼乐场景、历史建筑
- **音频资源**：经文朗读、古乐复原、专家讲解

### 3.3 AI 模型架构

| 功能模块 | 模型选型             | 规模/参数 | 推理环境        |
| -------- | -------------------- | --------- | --------------- |
| OCR 引擎 | TrOCR 微调           | 1.5B 参数 | 服务端/边缘计算 |
| 文言理解 | BERT-Chinese-Ancient | 340M      | 服务端          |
| 知识推理 | RAG+LLM              | 7B        | 服务端          |
| 内容生成 | 定制 LLaMA 微调      | 13B       | 服务端          |
| AR 识别  | EfficientNet-B3      | 12M       | 移动端          |

## 4. 开发路线图

### 4.1 阶段一：MVP（3 个月）

| 阶段     | 时间   | 关键功能                 | 里程碑       |
| -------- | ------ | ------------------------ | ------------ |
| 准备期   | 1-2 周 | 需求细化、技术栈确认     | 项目启动文档 |
| 原型开发 | 4 周   | UI 设计、基础框架搭建    | 可交互原型   |
| 核心功能 | 6 周   | OCR 基础识别、基本知识库 | 技术验证版本 |
| 集成测试 | 2 周   | 功能整合、Bug 修复       | MVP 内测版   |

#### 4.1.1 MVP 功能范围

- **古籍识别**：支持《论语》文本的基础 OCR
- **知识图谱**：完成《论语》前 5 章的知识点关联
- **AI 功能**：基于 GPT 模型的问答系统（无角色定制）
- **平台支持**：iOS 与 Android 基础版本

### 4.2 阶段二：成长期（6 个月）

| 功能模块         | 优先级 | 预计工时 | 技术挑战         |
| ---------------- | ------ | -------- | ---------------- |
| OCR 引擎升级     | 高     | 8 周     | 异体字识别准确率 |
| 《孟子》知识图谱 | 高     | 10 周    | 关系抽取自动化   |
| AI 角色定制      | 中     | 6 周     | 角色一致性       |
| 文言文转换器     | 中     | 6 周     | 多级别翻译质量   |
| AR 基础功能      | 低     | 8 周     | 3D 模型生成效率  |

### 4.3 阶段三：扩展期（持续）

- **内容扩展**：《大学》、《中庸》等经典纳入
- **社区功能**：用户注释、讨论、共创
- **教育版本**：针对学校、教育机构的定制版
- **开放平台**：API 开放给第三方开发者

### 9.4 设计风格指南

#### 9.4.1 配色方案

儒智 APP 采用传统雅致与现代简约相结合的配色体系，以营造既有文化底蕴又不失现代感的视觉体验。

**主色调**

- 主色：宣德青 `#1A5599` - 体现中国传统文化中的稳重与智慧
- 辅助色：书卷褐 `#A67E58` - 呈现古籍文献的历史质感
- 点缀色：点朱红 `#D73E35` - 用于强调和引导注意，如印章、标记等

**功能色**

- 正面反馈：青松绿 `#2A8D6A` - 成功状态、积极反馈
- 警示色：琉璃黄 `#FFB01D` - 提醒、警告信息
- 错误色：朱砂红 `#CF3A31` - 错误提示、删除操作

**中性色**

- 标题文字：墨玉黑 `#333333`
- 正文文字：青墨灰 `#666666`
- 辅助文字：淡墨灰 `#999999`
- 分割线/边框：云白灰 `#E6E6E6`
- 背景色：宣纸白 `#F9F7F2` - 轻微米黄色调，减少刺眼感
- 卡片背景：素白 `#FFFFFF`

**状态色变化**

- 按钮按压时透明度降低至 85%
- 链接悬停时加深 10% 色调

#### 9.4.2 字体系统

APP 采用结合传统书籍排版与现代可读性的字体系统。

**中文字体**

- 主标题：方正新书宋 (加粗)
- 副标题：思源宋体 CN Medium
- 正文内容：思源宋体 CN Regular
- 经典原文：方正楷体
- 注解/批注：方正仿宋

**英文/数字字体**

- 标题：Spectral SemiBold
- 正文：Spectral Regular
- 数据/时间：Spectral Light

**字号体系**

- 大标题：22px
- 页面标题：18px
- 副标题：16px
- 正文内容：15px
- 辅助文本：13px
- 注释文本：12px

**行高与段落**

- 标题行高：1.3 倍字号
- 正文行高：1.5 倍字号
- 段落间距：1.2 倍行高

#### 9.4.3 图标与视觉元素

**图标风格**

- 线框风格与填充风格结合
- 线条粗细适中，保持一致性
- 图标尺寸：24×24px (导航栏)、18×18px (功能图标)
- 具有传统文化元素的现代简约设计

**视觉元素**

- 传统纹样：用于分隔线、背景装饰
- 水墨效果：用于特殊场景背景、过渡动画
- 印章元素：用于认证、完成标记
- 宣纸纹理：用于背景、卡片底纹

**插图风格**

- 简约线条与传统国画元素结合
- 色彩与主配色方案保持一致
- 空场景插图体现传统文化场景

#### 9.4.4 交互元素

**按钮样式**

- 主要按钮：圆角矩形，填充主色
- 次要按钮：圆角矩形，细边框主色
- 文字按钮：无背景，主色文字
- 按钮高度：44px (主要)、36px (次要)、无固定高度 (文字)
- 状态变化：按下时轻微缩小并降低透明度

**表单元素**

- 输入框：浅色背景，底部细线边框
- 选择器：圆角矩形轮廓，选中项填充主色
- 开关：滑块设计，开启状态显示主色
- 单选/复选框：传统印章风格样式

**列表样式**

- 卡片式：白色背景，轻微阴影
- 分割线式：细线分隔，左侧留白
- 宫格式：均匀网格，适用功能快捷入口

**过渡与动效**

- 页面转场：平滑侧滑或淡入淡出
- 加载动画：水墨扩散效果
- 点击反馈：波纹效果
- 列表滚动：惯性滚动，平滑减速

**空状态与消息提示**

- 空状态页：结合传统元素的插图，简洁描述
- 轻提示：文字提示，从底部滑入
- 对话框：中心弹出，带有传统边框装饰

### 9.5 可访问性设计

- 颜色对比度符合 WCAG 2.0 AA 级标准
- 支持系统字体大小调节
- 关键操作区域触控面积不小于 44×44pt
- 提供语音辅助功能兼容性
- 重要内容不仅依赖颜色来传达
